My Process

  1. Investigation

    Before concept creation occurs I try to develop a greater understanding of the environment, audience, resources and potential opportunities related to the project.

  2. Analysis & Planning

    Requirements and goals are formulated. At this point user cases enhance the discovery of user needs. The types of technologies to use for building the project are explored. A project timeline is mapped out.

  3. Design

    Wireframes and Prototypes are created. These are then tested on usability, functionality and whether they meet all requirements or goals established.

  4. Development & Testing

    The functional website application is created following standards and best practices. A continual cycle of development and testing is done. Checks for cross-browser and device compatibility, and valid code are made.

  5. Launch

    The website goes through final optimization and then is pushed to a live state. Any additional maintenance and refinement will be done going forward.

Portfolio

New Resident Hub

Website that polls local residents on the quality of businesses in the area.

Built With

Meteor MongoDB NodeJS Blaze Bootstrap HTML CSS JavaScript

About

Users of local services would be able to vote on recommending multiple businesses by answering three quesions according to their personal experience. New residents would use the voting scores to make a quick and more informed decision for a business for their needs.

The application utilized real time updates to avoid additional page loads. Resulting in an improved user experience. Showing instant results from the database update from other user interactions is also likely to boost engagement. It reinforces the idea that a visitor should get involved in the interaction too.

Creating the New Resident Hub

Creating the project was a valuable experience because it provided some hands-on experience on building with node.js. I specifically chose to develop the project to learn a bit more about the full-stack process. Being able to understand the process will ultimately lead to a better constructed project.

On designing the site, I was able to get more comfortable with learning how to handle the interactions that occurred in the database. I also learned a great deal about the technologies used to create a web application with node.js. Command line basics was another new skill picked up while setting up a testing environment. I also took time to familiarize myself with the Meteor Framework, MongoDB, Git, the NPM package manager and Blaze templates.

Spa Delight

Website presence for a massage & facial spa small business.

Built With

HTML CSS JavaScript

About

Features include a contact form to increase communication to potential clients, modals, and animation. It's a simple and responsive design. Elements in the mobile version are kept minimal for a faster loading experience.

Creating Spa Delight

In creating the website I wanted the website the reflect a relaxing feel to better suit the type of business being represented. The clouds in the main background blues, greens, white, pastel yellow and pinks all fit a light and airy look.
